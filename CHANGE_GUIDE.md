# 本版修改说明

本版本已完成：
1. 删除首页顶部“默认空间 / 管理你的临期物品 / 邀请”这一块。
2. 点击关键词输入框右侧的扫码按钮后，识别结果会自动填入关键词输入框。
3. 新增物品页面里的“条形码&二维码 → 扫码”仍然会填入条码字段。
4. EdgeOne Makers 跳过 npm install / npm run build，直接部署 dist 目录。

上传 GitHub 时请上传本目录全部内容。
不要上传 node_modules、.edgeone、.env、.env.local。

5. 首页搜索框右侧已改为“搜索”按钮，不再使用旁边扫码。
6. 右侧悬浮扫码按钮已改成你截图那种四角扫描图标样式。
7. 如果未绑定 EdgeOne KV（变量名 EXPIRE_KV），现在会直接提示报错，不会再出现“提示成功但数据没有保存”的情况。


## 本版显示优化

- 首页现在会直接显示物品卡片，不需要先输入关键词。
- 前端默认用户 ID 固定为 `default`，避免更换浏览器、清除缓存后数据看不到。
- `/api/items` 现在会读取 KV 里所有 `item_` 开头的数据，兼容之前已经保存过的旧 userId 数据。
- KV 读取兼容更多返回格式，避免 KV 里有数据但列表为空。

## 七牛云与微信配置移入设置页

- 七牛云 AccessKey、SecretKey、Bucket、访问域名、上传域名现在在页面「设置」里填写。
- 微信公众号 AppID、AppSecret、模板ID、网站域名现在在页面「设置」里填写。
- 不使用图片上传或微信通知时可以留空。
- 环境变量仍可作为兜底，但不再要求在 EdgeOne Makers 中配置这些参数。


## 本次更新：前台设置 / 后台配置分离

- 纯域名访问：正常使用临期账本。
- `/admin/` 后缀访问：进入后台配置，用于七牛云、微信公众号、数据刷新、退出访问等配置。
- 纯域名里的「设置」改为九宫格样式，主要保留提醒设置、临期配置、数据导出。
- 新增「数据导出」页面，支持按物品状态、分类、标签、记录开始日期、记录结束日期筛选导出，不包含记录人。

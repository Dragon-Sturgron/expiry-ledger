# 本版修改说明

本版本已完成：
1. 删除首页顶部“默认空间 / 管理你的临期物品 / 邀请”这一块。
2. 点击关键词输入框右侧的扫码按钮后，识别结果会自动填入关键词输入框。
3. 新增物品页面里的“条形码&二维码 → 扫码”仍然会填入条码字段。
4. EdgeOne Makers 跳过 npm install / npm run build，直接部署 dist 目录。

上传 GitHub 时请上传本目录全部内容。
不要上传 node_modules、.edgeone、.env、.env.local。

5. 首页搜索框右侧已改为“搜索”按钮，不再使用旁边扫码。
6. 右侧悬浮扫码按钮已改成你截图那种四角扫描图标样式。
7. 如果未绑定 EdgeOne KV（变量名 EXPIRE_KV），现在会直接提示报错，不会再出现“提示成功但数据没有保存”的情况。


## 本版显示优化

- 首页现在会直接显示物品卡片，不需要先输入关键词。
- 前端默认用户 ID 固定为 `default`，避免更换浏览器、清除缓存后数据看不到。
- `/api/items` 现在会读取 KV 里所有 `item_` 开头的数据，兼容之前已经保存过的旧 userId 数据。
- KV 读取兼容更多返回格式，避免 KV 里有数据但列表为空。

## 七牛云与微信配置移入设置页

- 七牛云 AccessKey、SecretKey、Bucket、访问域名、上传域名现在在页面「设置」里填写。
- 微信公众号 AppID、AppSecret、模板ID、网站域名现在在页面「设置」里填写。
- 不使用图片上传或微信通知时可以留空。
- 环境变量仍可作为兜底，但不再要求在 EdgeOne Makers 中配置这些参数。


## 本次更新：前台设置 / 后台配置分离

- 纯域名访问：正常使用临期账本。
- `/admin/` 后缀访问：进入后台配置，用于七牛云、微信公众号、数据刷新、退出访问等配置。
- 纯域名里的「设置」改为九宫格样式，主要保留提醒设置、临期配置、数据导出。
- 新增「数据导出」页面，支持按物品状态、分类、标签、记录开始日期、记录结束日期筛选导出，不包含记录人。


## 本次更新：商品搜索选择、图片预览、分类配置

- 添加效期时，商品选择从下拉框改为可输入搜索的商品选择框。
- 商品较多且没有条码时，可输入商品名称、分类、条码、备注或标签进行搜索。
- 添加商品时，直接填写图片链接会显示图片预览。
- 设置页九宫格里的“操作记录”已替换为“分类配置”。
- 分类配置可维护常用商品分类，添加商品时可通过分类输入框选择。


## 本次更新：分类下拉、分类弹层、图片链接修复

- 添加商品的分类从输入框改为下拉框。
- 分类旁边新增“配置”按钮，可直接跳转分类配置。
- 分类配置页面改成弹层样式：标题、关闭按钮、搜索框、分类标签、添加分类。
- 添加商品时手动粘贴图片链接会用 `<img>` 直接预览，兼容带 `?x-oss-process=...&name=...` 的图片链接。
- 全局图片展示改为更稳的 `url("图片地址")` 写法，减少特殊字符导致背景图不显示的问题。


## 本次更新：分类样式与图片代理

- 添加商品里的分类不再使用原生下拉框，改为点击分类行进入分类弹层。
- 已去掉分类旁边的“配置”按钮。
- 分类弹层按截图样式优化，支持在搜索框输入分类名后直接点击“添加分类”。
- 图片链接预览改为经过 `/api/image/proxy` 代理加载，兼容带复杂参数的图片链接，并尽量规避外链防盗链导致的不显示。


## 本次更新：图片外链显示优化

- 图片预览优先使用原始图片链接直接显示。
- 图片预览增加 `referrerpolicy="no-referrer"`，避免部分 OSS/CDN 因来源页面 Referer 导致图片不显示。
- 如果直接显示失败，会自动降级到 `/api/image/proxy` 代理加载。
- 适合浏览器直接打开能看，但嵌入页面里不显示的图片链接。


## 本次更新：添加效期选择商品弹框

- 添加效期页面恢复为原来的简洁样式，不再在页面里直接显示商品列表。
- 点击“选择商品资料/选择”后弹出搜索框。
- 在弹框里输入关键词后才显示匹配商品。
- 例如商品有“商品A、商品B、螺蛳粉、宽粉”，输入“商品”只显示商品A/商品B，输入“粉”只显示螺蛳粉/宽粉。


## 本次更新：商品选择弹框与返回逻辑

- 选择商品弹框默认显示全部商品。
- 商品列表区域保持固定高度，商品过多时在区域内上下滑动。
- 输入关键词后按名称、条码、分类、备注、标签筛选商品。
- 所有左上角返回按钮改为返回上一页，而不是固定返回首页。
  - 首页 → 商品详情 → 返回：回首页
  - 添加效期 → 新增商品 → 返回：回添加效期


## 本次更新：选择商品弹框自适应宽度

- 选择商品弹框宽度改为按设备宽度自动伸缩。
- 手机小屏不会再超出屏幕边缘。
- 桌面端仍保持接近手机页面的最大宽度，不会横向铺满整个浏览器。
- 商品列表区域高度固定并支持上下滑动。


## 本次更新：底部按钮优化

- 已取消底部中间的“＋效期”按钮。
- 底部保留“＋商品”和“⚙设置”。
- 两侧按钮已设置为不换行，避免手机宽度较小时文字上下排版。
